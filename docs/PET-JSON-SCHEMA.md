## 4. What is **NOT** in `pet.json` (critical)

This manifest is **identity-only**. It contains **none** of the data a renderer
needs to actually animate the sheet. Specifically, all of the following are
**absent** and had to be derived from the pixels (see `docs/ASSET-INVENTORY.md`):

- ❌ Frame dimensions / cell size
- ❌ Grid rows & columns
- ❌ Frame counts per animation
- ❌ Animation names or clip definitions
- ❌ Frame order / sequencing
- ❌ Playback speed (fps) or per-frame duration
- ❌ Directional orientation metadata
- ❌ Anchor points / offsets / pivot
- ❌ Scale / DPI / logical size hints
- ❌ Any state → animation mapping

Because of this, the project introduces a **separate runtime manifest** that
adds exactly this missing layer without touching the original:

> `app/Sources/KhosrowApp/Resources/khosrow.runtime.json`
> (generated by `scripts/build_runtime_manifest.py`)

The runtime manifest **embeds the five `pet.json` identity fields verbatim** and
adds derived grid geometry, clip definitions, and the Claude-state map. This
keeps `pet.json` as the untouched identity source of truth while giving the app
a complete, testable animation description.

## 5. ChatGPT-specific metadata

- `spriteVersionNumber: 2` is the only ChatGPT-format-specific field. It signals
  the sheet-layout convention (a tiled grid of RGBA frames, rows = clips) that
  the host renderer is expected to know out-of-band. The **specific** grid
  dimensions for this pet are not encoded and vary per pet, which is why this
  project detects them empirically rather than hard-coding a spec.
- No telemetry, prompt, tool, account, or path-to-user data is present. The
  manifest is safe to commit and ship as-is.

## 6. Fields whose meaning is inferred

| Field | Inference | Basis |
|-------|-----------|-------|
| `spriteVersionNumber` | A format/version discriminator for the sheet layout, not a content version of this particular art. | Name + integer value; consistent with versioned asset formats. |
| `spritesheetPath` | Resolved **relative to the manifest file's directory** (both files sit at repo root). | Both files are siblings; the value is a bare filename. |

Everything else (`id`, `displayName`, `description`) is self-evident and used
verbatim.
